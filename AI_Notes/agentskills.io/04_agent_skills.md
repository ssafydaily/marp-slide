# Agent Skills
- AI 에이전트에 새로운 기능과 전문성을 부여하는 표준화된 방법.
- [오픈 표준](https://agentskills.io)으로, GitHub Copilot for VS Code, GitHub Copilot CLI, GitHub Copilot 클라우드 에이전트 등 여러 AI 에이전트에서 동작

----------------

## Agent Skills란?

- AI 에이전트의 기능을 특화된 지식과 워크플로우로 확장하기 위한 경량 오픈 포맷
- 스킬은 `SKILL.md` 파일을 담은 폴더
- 파일은 메타데이터(최소한 `name`과 `description`)와 에이전트가 특정 작업을 수행하는 방법을 알려주는 지침을 포함
- 스킬에는 스크립트, 참고 자료, 템플릿 등 다양한 리소스도 함께 묶을 수 있다

  ```
  my-skill/
  ├── SKILL.md          # 필수: 메타데이터 + 지침
  ├── scripts/          # 선택: 실행 가능한 코드
  ├── references/       # 선택: 문서
  ├── assets/           # 선택: 템플릿, 리소스
  └── ...               # 기타 추가 파일 또는 디렉터리
  ```

----------------

## 왜 Agent Skills인가?

- 에이전트의 능력은 날로 발전하고 있지만, 
- 실제 작업을 안정적으로 수행하는 데 필요한 맥락이 부족한 경우가 많다. 
- Skills는 절차적 지식과 회사·팀·사용자별 맥락을 에이전트가 필요할 때 불러오는 이식 가능한 버전 관리 폴더로 패키징함으로써 이 문제를 해결


> 덕분에 에이전트는 다음을 갖출 수 있다
> - **도메인 전문성**: 법률 검토 프로세스, 데이터 분석 파이프라인, 프레젠테이션 포맷팅 등 특화된 지식을 재사용 가능한 지침과 리소스로 습득
> - **반복 가능한 워크플로우**: 여러 단계의 작업을 일관되고 감사 가능한 절차로 전환
> - **제품 간 재사용**: 스킬을 한 번 만들면 Skills를 지원하는 모든 에이전트에서 사용

----------------

## Agent Skills는 어떻게 동작하는가?

- 에이전트는 **점진적 공개(progressive disclosure)** 방식으로 스킬을 세 단계에 걸쳐 로드:

1. **발견(Discovery)**: 시작 시, 에이전트는 각 스킬의 이름과 설명만 로드
    - 언제 관련이 있을지 파악하기에 충분한 최소한의 정보

2. **활성화(Activation)**: 작업이 스킬의 설명과 일치하면, 에이전트가 `SKILL.md`의 전체 지침을 컨텍스트로 읽어들인다

3. **실행(Execution)**: 에이전트가 지침을 따르며, 필요 시 번들된 코드를 실행하거나 참조 파일을 로드

- 전체 지침은 작업에 필요할 때만 로드되므로, 에이전트는 적은 컨텍스트 공간으로도 많은 스킬을 보유할 수 있다.

----------------

## 어디서 사용할 수 있나요?

- Agent Skills는 다수의 AI 도구 및 에이전트 클라이언트에서 지원
- Gemini CLI, Cursor, GitHub Copilot, Claude Code, OpenAI Codex 등 수십 개 이상의 제품이 이 포맷을 채택

# 명세서

> Agent Skills의 완전한 형식 명세서

## 디렉토리 구조

- 스킬은 최소한 `SKILL.md` 파일을 포함하는 디렉토리

```
skill-name/
├── SKILL.md          # 필수: 메타데이터 + 지침
├── scripts/          # 선택: 실행 가능한 코드
├── references/       # 선택: 문서
├── assets/           # 선택: 템플릿, 리소스
└── ...               # 추가 파일 또는 디렉토리
```

-----------------------------

## `SKILL.md` 형식

- `SKILL.md` 파일은 YAML 프론트매터(frontmatter)로 시작하고 이후 마크다운 본문이 작성되야 한다.

### 프론트매터

| 필드 | 필수 여부 | 제약 조건 |
|------|----------|----------|
| `name` | 필수 | 최대 64자. 소문자, 숫자, 하이픈만 허용. 하이픈으로 시작하거나 끝날 수 없음. |
| `description` | 필수 | 최대 1024자. 비어 있을 수 없음. 스킬이 무엇을 하는지, 언제 사용하는지 설명. |
| `license` | 선택 | 라이선스 이름 또는 번들 라이선스 파일 참조. |
| `compatibility` | 선택 | 최대 500자. 환경 요구 사항 명시 (대상 제품, 시스템 패키지, 네트워크 접근 등). |
| `metadata` | 선택 | 추가 메타데이터를 위한 임의 키-값 매핑. |
| `allowed-tools` | 선택 | 스킬이 사용할 수 있도록 사전 승인된 도구의 공백 구분 문자열. (실험적) |

--------------------------

**최소 예시:**

```markdown
---
name: skill-name
description: 이 스킬이 무엇을 하는지, 언제 사용하는지에 대한 설명.

---
```

**선택 필드 포함 예시:**

```markdown
---
name: pdf-processing
description: PDF 텍스트 추출, 양식 작성, 파일 병합. PDF 처리 시 사용.
license: Apache-2.0
metadata:
  author: example-org
  version: "1.0"

---
```

------------------------------

#### `name` 필드 : 필수

* 1~64자여야 함
* 유니코드 소문자 영숫자(`a-z`, `0-9`)와 하이픈(`-`)만 허용
* 하이픈으로 시작하거나 끝날 수 없음, 연속된 하이픈(`--`) 불가
* 상위 디렉토리 이름과 일치해야 함

**유효한 예시:**
```yaml
name: pdf-processing
name: data-analysis
name: code-review
```

**유효하지 않은 예시:**
```yaml
name: PDF-Processing  # 대문자 불가
name: -pdf            # 하이픈으로 시작 불가
name: pdf--processing # 연속 하이픈 불가
```

----------------

#### `description` 필드 : 필수

* 1~1024자여야 함
* 스킬이 무엇을 하는지, 언제 사용하는지 모두 설명해야 함
* 에이전트가 관련 작업을 식별하는 데 도움이 되는 키워드 포함 권장

**좋은 예시:**
```yaml
description: PDF 파일에서 텍스트와 표를 추출하고, PDF 양식을 작성하며, 여러 PDF를 병합합니다. 
PDF 문서 작업 시 또는 사용자가 PDF, 양식, 문서 추출을 언급할 때 사용하세요.
```

**좋지 않은 예시:**
```yaml
description: PDF 관련 도움을 줍니다.
```

---------------------

#### `license` 필드 : 선택

* 스킬에 적용된 라이선스를 지정, 간결하게 유지 권장 (라이선스 이름 또는 번들 라이선스 파일명)

#### `compatibility` 필드 : 선택

* 제공 시 1~500자여야 함
* 스킬에 특정 환경 요구 사항이 있는 경우에만 포함
* 대상 제품, 필요한 시스템 패키지, 네트워크 접근 필요 여부 등을 명시 가능

**예시:**
```yaml
compatibility: Claude Code(또는 유사 제품)용으로 설계됨
compatibility: git, docker, jq 및 인터넷 접근 필요
compatibility: Python 3.14 이상 및 uv 필요
```

> 대부분의 스킬은 `compatibility` 필드가 필요하지 않습니다.

----------------------

#### `metadata` 필드 : 선택

* 문자열 키에서 문자열 값으로의 매핑
* 클라이언트가 Agent Skills 명세에 정의되지 않은 추가 속성을 저장할 때 사용
* 우발적인 충돌을 피하기 위해 키 이름을 충분히 고유하게 지정 권장

#### `allowed-tools` 필드 : 선택

* 사전 승인된 실행 도구를 공백으로 구분한 문자열
* ***실험적 기능***. 에이전트 구현에 따라 지원 여부가 다를 수 있음

------------------

### 본문 내용

- 프론트매터 이후의 마크다운 본문에는 스킬 지침을 포함
- 형식 제한은 없으며, 에이전트가 작업을 효과적으로 수행하는 데 도움이 되는 내용을 자유롭게 작성

> **권장 섹션:**
> 
> * 단계별 지침
> * 입출력 예시
> * 일반적인 예외 케이스

- 에이전트는 스킬을 활성화하면 이 파일 전체를 불러온다
- 내용이 길다면 별도 파일로 분리하는 것을 고려한다.

--------------------

## 선택적 디렉토리

### `scripts/`

- 에이전트가 실행할 수 있는 코드를 포함:

  * 독립적으로 실행되거나 의존성을 명확히 문서화해야 함
  * 유용한 오류 메시지를 포함해야 함
  * 예외 상황을 적절히 처리해야 함

- 지원 언어는 에이전트 구현에 따라 다릅니다. 일반적으로 Python, Bash, JavaScript가 지원

----------------

### `references/`

- 에이전트가 필요 시 참조할 수 있는 추가 문서를 포함합니다:

  * `REFERENCE.md` - 상세 기술 참조
  * `FORMS.md` - 양식 템플릿 또는 구조화된 데이터 형식
  * 도메인별 파일 (`finance.md`, `legal.md` 등)

- 각 참조 파일은 특정 주제에 집중하도록 유지
- 에이전트는 필요할 때만 이 파일들을 불러오므로, 파일이 작을수록 컨텍스트 사용이 줄어든다.

-----------------

### `assets/`

- 정적 리소스를 포함:

  * 템플릿 (문서 템플릿, 설정 템플릿)
  * 이미지 (다이어그램, 예시)
  * 데이터 파일 (조회 테이블, 스키마)

------------------------

## 점진적 공개 (Progressive Disclosure)

- 에이전트는 스킬을 *점진적으로* 불러오며, 작업에 필요한 경우에만 더 많은 세부 정보를 가져온다. 
- 스킬은 이 방식을 활용하도록 구성되어야 한다.

1. **메타데이터** (약 100 토큰): 시작 시 모든 스킬의 `name`과 `description` 필드가 로드됨
2. **지침** (5000 토큰 이하 권장): 스킬 활성화 시 `SKILL.md` 전체 본문이 로드됨
3. **리소스** (필요 시): `scripts/`, `references/`, `assets/` 등의 파일은 필요할 때만 로드됨

- `SKILL.md`는 **500줄 이하로 유지** 하고, 상세 참조 자료는 별도 파일로 분리하시오.


--------------------

## 파일 참조

- 스킬 내 다른 파일을 참조할 때는 스킬 루트에서의 상대 경로를 사용:

  ```markdown
  자세한 내용은 [참조 가이드](references/REFERENCE.md)를 참고하세요.

  추출 스크립트 실행:
  scripts/extract.py
  ```

- `SKILL.md`에서 파일 참조는 한 단계 깊이까지만 유지한다. 깊게 중첩된 참조 체인은 피한다.

## 유효성 검사

[skills-ref](https://github.com/agentskills/agentskills/tree/main/skills-ref) 참조 라이브러리를 사용하여 스킬을 검증:

```bash
skills-ref validate ./my-skill
```

- 이 명령은 `SKILL.md` 프론트매터가 유효하고 모든 명명 규칙을 따르는지 확인합니다.
