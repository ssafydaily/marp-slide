# 코드 베이스 분석

## 1. 프로젝트 열기
- CLI: 탐색할 프로젝트 디렉토리에서 Claude Code를 실행
- 윈도우 앱: `Code` 탭에서 폴더 선택


## 2. 전체 구조 파악
- Claude가 디렉토리 구조를 탐색하고, README, package.json 등을 분석하여 프로젝트 개요를 정리

```
예시>
이 프로젝트의 구조를 설명해줘.
주요 기능, 아키텍처 패턴, 어떤 문제를 해결하는 프로젝트인지 알려줘.
```
- 코드를 읽고 분석하는 작업이라 `Shift + Tab` 으로 **Auto-Accept** 모드로 전환해서 작업

----------------

## 코드 분석

- 특정 기능이 어떻게 동작하는지 추적하기

```
예시>
@run_agent.py 를 실행하면 내부적으로 어떤 일이 일어나는지 코드 흐름을 처음부터 끝까지 추적해줘.
```

## 함수 호출 추적

- 함수가 프로젝트 내에서 어떻게 사용되는지 추적

```
예시>
이 함수가 어디서 호출되는지 찾아줘: [핵심 함수명]
호출 체인을 보여줘.
```
-------------------

## 온보딩 문서 작성

```
예시>
지금까지 분석한 내용을 바탕으로 새 팀원을 위한 온보딩 문서를 만들어줘.
포함할 내용:
1. 프로젝트 개요 (2~3줄)
2. 디렉토리 구조와 각 폴더 역할
3. 핵심 파일 5개와 역할
4. 주요 코드 흐름
5. 개발 시작하기 (설치, 실행, 테스트)

ONBOARDING.md 파일로 저장해줘.
```
-----------------
## 주의 사항

- 전체 → 구조 → 세부 순서로 탐색합니다. 세부 코드부터 보면 길을 잃습니다.
- 큰 프로젝트(10만 줄 이상)는 /clear로 세션을 정리하면서 영역별로 나누어 탐색합니다.
- Claude가 추측하는 부분이 있으면 실제 코드를 열어서 검증합니다.

# 파일 지정해서 지시하기

- 단순히 "이런 기능을 수정해줘" 락 하면 Claude가 관련 파일을 찾는 노력이 필요하다.
- `@파일명` 으로 알려주면 바로 파일을 열고 작업한다.

## 자동 완성
- 프롬프트에 `@`을 입력하면 파일 목록이 나타난다.

```
@store/index.js 이 파일이 뭘 하는지 알려줘
```
------------

## 정확하게 지시하기
```
로그인 함수에 이메일 검증 추가해줘
```

```
@auth/login.py의 authenticate 함수에 이메일 형식 검증을 추가해줘.
정규식으로 @와 도메인이 포함되었는지 확인하고,
유효하지 않으면 ValueError("Invalid email format")를 발생시켜줘.

```

**정확한 지시의 구성요소**

| 요소 |	예시 |	효과 |
| ---- | --- | ----- |
| 파일 |	@auth/login.py |	어떤 파일을 수정할지 확정 |
| 위치 |	authenticate 함수 |	파일 안에서 어디를 수정할지 확정 |
| 결과 |	ValueError("Invalid email format") |	원하는 동작을 구체적으로 명시 |

---------------

```
@routes/auth_routes.py의 /login 엔드포인트에서도 이 파라미터를 전달하도록 수정해줘.
```
<div class="callout tip">
  <div class="callout-title">

  참고 사항

  - 프로젝트가 매우 클 때(수천 개 파일)는 @로 범위를 좁혀주는 것이 특히 중요하다.
