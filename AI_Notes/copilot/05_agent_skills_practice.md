## 스킬 만들기
- 스킬은 `SKILL.md` 파일이 담긴 폴더
- VS Code는 기본적으로 `.agents/skills/` 경로에서 스킬을 탐색
- 프로젝트에 `.agents/skills/roll-dice/SKILL.md` 파일을 생성

  ````markdown
  ---
  name: roll-dice
  description: 난수 생성기를 사용해 주사위를 굴립니다. 주사위 굴리기(d6, d20 등)나
          무작위 주사위 결과 생성을 요청받을 때 사용하세요.
  ---
  주사위를 굴리려면, 1부터 지정한 면의 수까지 난수를 생성하는 아래 명령어를 사용하세요:

  ```bash
  echo $((RANDOM % <면의 수> + 1))
  ```
  ```powershell
  Get-Random -Minimum 1 -Maximum (<면의 수> + 1)
  ```
  `<면의 수>` 부분을 주사위의 면 수로 교체하세요 (예: 일반 주사위는 6, d20은 20).
  ````
------------------------------


* **`name`** — 스킬의 짧은 식별자. 폴더 이름과 일치해야 합니다.
* **`description`** — 에이전트에게 이 스킬을 언제 사용할지 알려줍니다. 에이전트가 스킬 활성화 여부를 판단하는 기준입니다.
* **본문** — 스킬이 활성화되었을 때 에이전트가 따르는 지침. 여기서는 사용자의 요청에서 면의 수를 대입해 터미널 명령어로 난수를 생성하도록 지시합니다.

## 직접 실행해보기

1. VS Code에서 프로젝트를 엽니다.
2. Copilot Chat 패널을 엽니다.
3. 채팅 패널 하단의 모드 드롭다운에서 **에이전트(Agent)** 모드를 선택합니다.
4. `/skills`를 입력해 `roll-dice`가 목록에 표시되는지 확인합니다. 표시되지 않으면 파일이 프로젝트 루트 기준으로 `.agents/skills/roll-dice/SKILL.md` 경로에 있는지 확인하세요.
5. **"d20을 굴려줘"** 라고 입력합니다.

에이전트가 `roll-dice` 스킬을 활성화해야 합니다. 터미널 명령어 실행 허가를 요청할 수 있으니 허용해 주세요. 명령어를 실행한 후 1에서 20 사이의 난수를 반환합니다.

> **참고:** 도구 사용 신뢰도는 모델마다 다릅니다. 일부 모델은 스킬 지침에 따라 명령어를 일관되게 실행하지만, 그렇지 않은 모델도 있습니다. 에이전트가 터미널 명령어 없이 응답한다면, 모델 드롭다운에서 다른 모델을 선택해 보세요.

## 동작 원리

뒤에서 일어난 일을 단계별로 살펴봅니다:

1. **탐색(Discovery)** — 채팅 세션이 시작되면 에이전트가 기본 스킬 디렉토리를 스캔해 스킬을 발견합니다. 이때는 `name`과 `description`만 읽어, 스킬이 관련성이 있는지 판단할 수 있는 최소한의 정보만 로드합니다.
2. **활성화(Activation)** — 주사위 굴리기를 요청받으면, 에이전트가 질문을 스킬의 description과 매칭해 `SKILL.md` 본문 전체를 컨텍스트에 로드합니다.
3. **실행(Execution)** — 에이전트가 본문의 지침에 따라, 요청된 면의 수에 맞게 터미널 명령어를 조정해 실행합니다.

이 과정은 **점진적 공개(progressive disclosure)** 방식을 활용해, 모든 스킬의 지침을 처음부터 로드하지 않고도 에이전트가 다수의 스킬에 접근할 수 있게 합니다.
