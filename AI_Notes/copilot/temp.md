### 디자인 미리보기 및 개선하기

Agents 창은 구체적인 코드 하나하나를 검토하기보다, 에이전트에게 작업을 완전히 맡긴 후 그 결과물을 검증하는 워크플로우에 매우 유용합니다. 웹 기반 애플리케이션의 경우, VS Code를 벗어나지 않고도 내장 브라우저(Integrated Browser)를 통해 에이전트가 작업한 결과를 바로 확인할 수 있습니다.

생성된 포트폴리오를 내장 브라우저로 미리 보려면 다음 단계를 따르세요:

1. **Files(파일)** 패널에 에이전트가 생성한 파일 목록이 표시됩니다. `index.html` 파일을 우클릭한 뒤 Open in Integrated Browser(내장 브라우저에서 열기)를 선택합니다.
내장 브라우저로 여는 옵션이 보이지 않는다면, 현재 **Files** 패널 안에 있는지 다시 한번 확인하세요.
> [!TIP]
> 모달 창 타이틀 바에 있는 **Open in Editor Area(에디터 영역에서 열기)** 버튼을 누르면 채팅 창 옆에 브라우저를 나란히 띄워 두고 볼 수 있습니다.


2. 페이지 디자인을 일부 수정해 보겠습니다. 내장 브라우저에서 **Add Element to Chat(채팅에 요소 추가)** 버튼을 선택하여 선택 모드로 들어갑니다.
3. 페이지 위로 마우스를 올려 바꾸고 싶은 요소(예: 메인 제목)를 선택합니다.
그러면 에이전트가 선택된 요소의 HTML, CSS 및 스크린샷을 프롬프트의 맥락(Context)으로 자동 추가합니다.
4. 채팅 입력창에 원하는 변경 사항을 설명하는 프롬프트를 입력하고 `kbstyle(Enter)`를 누릅니다. 예시:
```prompt

```



Use a gradient color for the text and use cursive.
(텍스트에 그라디언트 색상을 적용하고 필기체(cursive)로 변경해줘.)

```

5. 에이전트가 선택한 요소에 변경 사항을 적용합니다. 내장 브라우저에서 페이지를 새로고침하여 업데이트된 모습을 확인하세요.

> 내장 브라우저 덕분에 다른 창으로 전환할 필요 없이 에이전트의 작업물을 즉시 확인하고 다듬을 수 있습니다.

### 변경 사항 검토 및 커밋하기

에이전트가 작업한 내용을 최종 커밋하기 전에, 실제로 어떤 코드 변경이 일어났는지 검토하고 싶을 수 있습니다. **Changes(변경 사항)** 패널에는 에이전트가 세션 동안 생성하거나 수정한 모든 파일이 나열됩니다. 파일 변경 사항을 검토하고 커밋하려면 다음 단계를 따르세요:

1. **Changes** 패널을 선택하여 에이전트가 추가하거나 수정한 파일 목록을 확인합니다. 각 항목에는 변경 통계 정보와 추가/삭제/수정 표시가 함께 나타납니다.

    ![에이전트가 변경한 파일 목록을 보여주는 Agents 창의 Changes 패널 스크린샷.](images/getting-started/changes-panel.png)

    참고로 세션 목록의 항목에서도 이러한 변경 통계 요약을 확인할 수 있습니다.

2. 파일을 선택하면 디프(Diff) 뷰가 열려 에이전트가 수정한 코드를 상세히 검토할 수 있습니다. 타이틀 바의 탐색 컨트롤을 이용해 여러 파일을 오갈 수 있습니다.

    ![에이전트가 파일에 적용한 변경 사항을 보여주는 디프 뷰 스크린샷.](images/getting-started/diff-view.png)

    이번 케이스에서는 모든 파일이 새로 생성되었으므로 디프 뷰의 모든 줄이 추가(Additions)로 표시됩니다. 기존 파일이 수정된 경우에는 추가된 코드와 삭제된 코드가 함께 표시됩니다.

    > [!TIP]
    > 디프 뷰에서 특정 코드 블록을 선택하면, 코드의 해당 부분에 대해 에이전트에게 즉시 인라인 피드백을 보낼 수 있습니다.

3. 디프 뷰를 닫고 **Changes** 패널에서 **Commit Changes(변경 사항 커밋)**를 선택하여 에이전트의 작업물을 Git 저장소에 저장합니다.

    변경 사항을 커밋하고 나면 대기 중인 변경 사항이 없으므로 **Changes** 패널이 다시 빈 상태로 돌아갑니다. 세션 목록에 표시되던 변경 통계 숫자도 함께 사라집니다.

## 에디터에서 에이전트와 함께 코드 작성하기

<div class="docs-action" data-show-in-doc="false" data-show-in-sidebar="true" title="Chat 뷰 탐색하기">
에디터와 나란히 Chat 뷰를 띄워두고, 활성화된 작업 공간에서 에이전트의 도움을 받아 코딩 작업을 진행해 보세요.

* [Chat 뷰에 대해 알아보기](/docs/agents/chat-view.md)

</div>

어떤 변경 작업은 코드를 중심에 두고, 내가 코드를 직접 작성하면서 에이전트의 보조를 받는 방식을 선호할 수 있습니다. 예를 들어, 다크 모드/라이트 모드 테마 전환 버튼을 추가하면서 스타일을 세밀하게 조정해 나가는 경우가 그렇습니다. 이 접근 방식을 위해 에디터 화면으로 전환하고 **Chat(채팅) 뷰**를 사용해 보겠습니다.

### 작업 공간용 에디터 열기

1. Agents 창의 타이틀 바에 있는 **Open in Editor(에디터에서 열기)** 버튼을 선택하여 현재 활성화된 작업 공간을 일반 에디터 창으로 엽니다.

    ![Agents 창 타이틀 바의 Open in Editor 버튼 스크린샷.](images/getting-started/open-in-editor-button.png)

    그러면 해당 작업 공간이 열린 새 VS Code 창이 나타납니다. 오른쪽 사이드바에는 Chat 뷰가 그대로 유지되므로, 에디터에서 작업하는 동안에도 에이전트와 계속 소통할 수 있습니다.

2. 왼쪽 사이드바에 프로젝트 파일들을 보여주는 **탐색기(Explorer)** 뷰가 나타난 것을 볼 수 있습니다. 파일을 선택하면 메인 영역의 에디터 탭에 해당 파일이 열립니다.

    ![포트폴리오 파일이 열린 탐색기 뷰와 이전 에이전트 세션이 이어지는 Chat 뷰가 함께 표시된 에디터 스크린샷.](images/getting-started/explorer-and-chat-view.png)

    오른쪽 사이드바의 Chat 뷰에는 방금 전 Agents 창에서 생성하여 진행 중이던 에이전트 세션이 그대로 이어집니다.

### Chat 뷰에서 새 세션 시작하기

Chat 뷰는 에디터 탭 바로 옆인 오른쪽 사이드바에 위치하며, 사용자가 코드를 작성하는 동안 에이전트가 긴밀하게 보조할 수 있도록 최적화되어 있습니다.

이 단계에서는 포트폴리오 페이지에 테마 전환 버튼을 추가하는 에이전트를 실행하기 위해 새 세션을 시작합니다. 에이전트가 파일에 변경 사항을 직접 적용하면, 여러분은 에디터 내에서 인라인 디프(Inline Diff) 형태로 실시간 검토할 수 있습니다.

1. **New Chat(새 채팅)** (`+`) 버튼을 선택하여 새 세션을 시작합니다.

    ![Chat 뷰 타이틀 바의 New Chat 버튼 스크린샷.](images/getting-started/new-chat-button.png)

2. **Session Target(세션 대상)** 드롭다운에서 **Local(로컬)**이 선택되어 있는지 확인합니다. 그래야 에이전트가 현재 에디터 맥락 안에서 파일, 도구, 내장 브라우저에 접근하여 작동할 수 있습니다.

    ![Local 옵션이 선택된 Chat 뷰의 Session Target 드롭다운 스크린샷.](images/getting-started/session-target-dropdown.png)

3. 채팅 입력창에 다음 프롬프트를 입력하고 `kbstyle(Enter)`를 누릅니다:

    ```prompt
Add a theme switcher button that toggles between a light and dark color theme for the page.
(페이지의 라이트 모드와 다크 모드를 전환할 수 있는 테마 전환 버튼을 추가해줘.)

```

```
에이전트가 파일에 변경 사항을 적용하기 시작합니다. 코드가 에디터 안으로 실시간 스트리밍되며 인라인 디프로 표시되는 것을 볼 수 있습니다.

```

4. Chat 뷰에 변경된 파일 목록이 나타납니다. 파일을 열어 에디터에서 변경 사항을 직접 확인하세요. 에디터에 오버레이로 표시되는 컨트롤을 사용해 개별 수정 사항을 Keep(수락)하거나 Undo(되돌리기)할 수 있습니다.
변경 사항을 받아들이려면 **Keep**을 선택합니다.
5. `index.html` 파일을 선택한 후, 타이틀 바에 있는 **Open in Integrated Browser(내장 브라우저에서 열기)** (지구본 모양) 버튼을 눌러 새로 추가된 테마 전환 버튼이 잘 작동하는지 브라우저에서 미리 확인합니다.
6. 이번에는 에이전트에게 직접 내장 브라우저를 통해 페이지를 미리 보고 새 기능이 잘 작동하는지 검증하라고 요청해 보겠습니다. 이렇게 하면 에이전트가 브라우저에 렌더링된 시각적 결과를 바탕으로 코드를 스스로 보완할 수 있습니다. 다음 프롬프트를 입력하고 `kbstyle(Enter)`를 누르세요:
```prompt

```



Verify that the theme switcher works correctly and review the design aligns with the rest of the page. If there are any issues, fix them.
(테마 전환 버튼이 올바르게 작동하는지 검증하고, 디자인이 페이지의 나머지 부분과 잘 어우러지는지 검토해줘. 문제가 있다면 수정해줘.)

```

    ![에이전트가 변경 사항을 미리 보고 검증하기 위해 내장 브라우저를 여는 모습을 담은 스크린샷.](images/getting-started/agent-validate-changes.png)

    에이전트가 내장 브라우저를 열어도 되는지 승인을 요청합니다. 에이전트가 브라우저에 접근하여 변경 사항을 검증할 수 있도록 **Allow in this session(이 세션에서 허용)**을 선택합니다.

축하합니다! 에이전트 중심(Agent-first) 방식과 코드 중심(Code-first) 방식을 모두 활용하여 AI 에이전트와 함께 포트폴리오 페이지를 성공적으로 빌드했습니다. 또한, 내장 브라우저를 통해 에이전트가 스스로의 변경 사항을 직접 미리 보고 검증하도록 만드는 방법도 배워보았습니다.
