자바스크립트에서 클래스 없이 객체를 생성할 수 있는 이유는 자바스크립트가 **프로토타입 기반(Prototype-based) 언어**이기 때문입니다.

전통적인 클래스 기반 언어(Java, C++ 등)에서 클래스가 '객체를 만들기 위한 설계도'라면, 자바스크립트에서 객체는 '설계도 없이 바로 존재하는 실체'에 가깝습니다.

---

### 1. 프로토타입 기반 철학

자바스크립트는 모든 것이 객체로부터 시작합니다. 클래스라는 추상적인 개념을 먼저 정의하고 인스턴스를 찍어내는 방식이 아니라, **이미 존재하는 객체를 복사하거나(Cloning) 새로운 속성을 즉석에서 추가**하는 방식을 취합니다.

### 2. 객체 리터럴 (Object Literal)

자바스크립트에서 가장 간단하게 객체를 만드는 방법은 중괄호 `{}`를 사용하는 것입니다. 엔진 내부적으로는 클래스 정의 없이도 메모리 공간을 확보하고 키-값 쌍을 저장할 수 있도록 설계되어 있습니다.

```javascript
const user = {
  name: 'Gemini',
  greet: function() { console.log('Hello!'); }
};

```

위 코드는 별도의 클래스 선언 없이도 즉시 실행 가능한 객체를 생성합니다.

### 3. 동적 타이핑과 동적 속성 추가

자바스크립트는 **동적 언어**입니다. 객체가 생성된 이후에도 언제든지 속성을 추가하거나 삭제할 수 있습니다.

* 클래스 기반 언어: 객체의 구조가 클래스에 고정되어 있어 런타임에 속성을 마음대로 추가하기 어렵습니다.
* 자바스크립트: 객체가 일단 생성되면, 그 객체는 자신만의 독립적인 저장 공간을 가집니다. 따라서 실행 중에 `user.age = 25;`와 같이 새로운 데이터를 밀어넣는 것이 가능합니다.

### 4. 프로토타입 체인 (Prototype Chain)

클래스가 없어도 객체 간의 '상속'과 '공유'가 가능한 이유는 **프로토타입** 덕분입니다. 모든 객체는 자신의 부모 역할을 하는 `[[Prototype]]` 링크를 가집니다.

* 특정 객체의 속성을 찾을 때 없으면, 연결된 부모(프로토타입) 객체를 거슬러 올라가며 찾습니다.
* 이러한 메커니즘 덕분에 굳이 클래스 계층 구조를 미리 짜놓지 않아도 객체들끼리 기능을 공유할 수 있습니다.

### 5. 그러면 ES6 `class`는 무엇인가요?

2015년에 도입된 `class` 키워드는 사실 새로운 동작 방식을 만드는 것이 아니라, 기존의 프로토타입 기반 동작을 클래스처럼 보이게 만든 '문법적 설탕(Syntactic Sugar)'입니다. 겉모습은 클래스지만 내부적으로는 여전히 프로토타입 방식으로 돌아갑니다.

---

**요약하자면:**
자바스크립트는 "설계도(Class)가 있어야 물건(Object)을 만든다"는 고정관념을 깨고, "물건을 먼저 만들고 필요하면 수정하거나 다른 물건의 특징을 빌려온다"는 유연한 철학을 가지고 있기 때문에 클래스 없이도 객체 생성이 가능합니다.
