# 답변

자바스크립트에서 프로토타입(Prototype)은 객체 간의 상속을 구현하기 위해 사용되는 메커니즘입니다. 자바스크립트의 모든 객체는 부모 역할을 하는 객체와 연결되어 있으며, 이 부모 객체를 '프로토타입'이라고 부릅니다.

---

## 1. 핵심 개념: 유전자(Prototype)

쉽게 비유하자면 프로토타입은 '부모가 자식에게 물려주는 유전자'와 같습니다.

* **상속**: 자식 객체는 부모(프로토타입)가 가진 기능을 자유롭게 가져다 쓸 수 있습니다.
* **공유**: 여러 자식 객체가 하나의 프로토타입을 공유하므로, 메모리를 효율적으로 사용할 수 있습니다.

---

## 2. 두 가지 중요한 속성

프로토타입을 이해하려면 아래 두 가지 용어를 구분해야 합니다.

### ① `prototype` 속성 (함수만 가짐)

함수(정확히는 생성자 함수)가 생성될 때 자동으로 부여되는 속성입니다. 이 함수를 통해 만들어질 **자식 객체들이 물려받을 청사진**을 담는 장소입니다.

```javascript
function Person(name) {
  this.name = name;
}

// Person의 'prototype' 주머니에 메서드를 저장
Person.prototype.sayHello = function() {
  console.log(`Hi, I'm ${this.name}`);
};

```

### ② `__proto__` 속성 (모든 객체가 가짐)

모든 객체가 생성될 때 가지게 되는 속성으로, **나를 만든 부모의 `prototype`을 가리키는 비밀 링크**입니다.

```javascript
const kim = new Person('Kim');

console.log(kim.__proto__ === Person.prototype); // true

```

`kim`이라는 객체 안에는 `sayHello`가 없지만, `kim.__proto__`가 `Person.prototype`을 가리키고 있기 때문에 메서드를 호출할 수 있는 것입니다.

---

## 3. 프로토타입 체인 (Prototype Chain)

객체의 속성을 찾을 때 **"자신 -> 부모 -> 조부모..."** 순으로 거슬러 올라가며 탐색하는 과정을 말합니다.

1. `kim.toString()`을 호출합니다.
2. `kim` 객체에 `toString`이 있는지 확인합니다. (없음)
3. 부모인 `Person.prototype`에 있는지 확인합니다. (없음)
4. 조부모인 `Object.prototype`에 있는지 확인합니다. (있음!)
5. 실행합니다.

만약 최상위인 `Object.prototype`에서도 찾지 못하면 `null`을 반환하며 탐색이 종료됩니다.

---

## 4. 프로토타입을 사용하는 이유: 메모리 절약

만약 프로토타입을 쓰지 않고 생성자 함수 안에 메서드를 직접 넣으면 어떻게 될까요?

```javascript
function Person(name) {
  this.name = name;
  this.sayHello = function() { ... }; // 객체를 만들 때마다 함수가 새로 생성됨
}

```

객체를 10,000개 만들면 `sayHello` 함수도 10,000개가 메모리에 할당됩니다. 하지만 **프로토타입**에 메서드를 등록하면, 함수는 단 1개만 존재하고 10,000개의 객체가 이를 공유해서 사용하므로 성능과 메모리 면에서 압도적으로 유리합니다.

---

## 5. 결론

* **Prototype**: 자식들에게 물려줄 속성들을 모아놓은 **공통 창고**.
* **proto**: 내 부모 창고가 어디인지 알려주는 **이정표**.
* **Prototype Chain**: 이 이정표를 따라 올라가며 기능을 찾는 **탐색 과정**.

자바스크립트에서 "모든 것은 객체다"라고 말하는 이유는, 결국 숫자나 문자열 같은 기본 자료형조차 각각의 프로토타입(Number.prototype, String.prototype)을 가지고 있어 그 안의 메서드들을 빌려 쓸 수 있기 때문입니다.

이 프로토타입 개념이 이해되셨다면, 이제 자바스크립트의 상속과 `class` 문법의 동작 원리를 완벽히 파악하신 것입니다!
