결론부터 말씀드리면, **`prototype` 속성은 함수(Function) 객체만 가지고 있는 특별한 속성**이며, 일반적인 객체에는 존재하지 않습니다. 하지만 모든 객체는 자신의 부모를 가리키는 **`[[Prototype]]`이라는 내부 링크**를 가지고 있습니다.

이 차이를 명확히 구분하는 것이 프로토타입의 핵심입니다.

---

### 1. `prototype` 속성은 누구에게 있는가?

자바스크립트에서 `prototype`이라는 이름의 **직접적인 속성**은 오직 **'생성자 함수' 역할을 할 수 있는 함수 객체**만 가집니다.

* **Class로 정의한 경우**: 클래스 역시 내부적으로는 함수이므로 `prototype` 속성을 가집니다.
* **일반 함수로 정의한 경우**: `function Person() {}`과 같이 정의하면 이 함수도 `prototype` 속성을 가집니다.
* **일반 객체**: `{ name: 'Kim' }`처럼 생성된 일반 객체는 `prototype`이라는 속성을 가지고 있지 않습니다.

---

### 2. 클래스 없이 생성된 객체에는 정말 없는가?

여기서 혼동하기 쉬운 두 가지를 구분해야 합니다.

#### ① `prototype` 속성 (Property)

일반 객체에는 없습니다.

```javascript
const obj = { a: 1 };
console.log(obj.prototype); // undefined

```

#### ② `[[Prototype]]` 내부 슬롯 (Internal Slot)

이것은 클래스로 만들었든, 리터럴 `{}`로 만들었든 **모든 객체가 가지고 있는 숨겨진 속성**입니다. 브라우저 개발자 도구에서는 `[[Prototype]]`으로 표시되며, 코드상에서는 `__proto__`를 통해 접근할 수 있습니다.

```javascript
const obj = { a: 1 };
console.log(obj.__proto__); // Object.prototype 출력 (부모는 있음!)

```

---

### 3. 왜 함수(클래스)에만 `prototype` 속성이 있나요?

`prototype` 속성의 존재 목적은 "나를 통해 태어날 아이들이 쓸 공용 창고"를 미리 준비해두는 것이기 때문입니다.

1. **함수(설계도)**: "나중에 누가 나를 `new`로 호출해서 자식을 만들면, 내 `prototype`에 있는 것들을 다 물려줘야지!" 하고 미리 창고를 준비합니다.
2. **객체(결과물)**: "나는 이미 태어난 실체야. 나를 만든 부모의 창고가 어디인지만 알면 돼(`__proto__`). 내가 직접 창고(`prototype`)를 관리할 필요는 없어."

---

### 4. 한눈에 비교하기

| 구분 | `prototype` 속성 보유 여부 | `__proto__` (부모 링크) 보유 여부 |
| --- | --- | --- |
| **생성자 함수 / Class** | **Yes** | Yes |
| **일반 객체 (리터럴 `{}`)** | **No** | Yes |
| **배열, 숫자 등 인스턴스** | **No** | Yes |

### 요약

클래스나 함수가 아닌 **일반 객체에는 `prototype`이라는 속성이 없습니다.** 하지만 그 객체를 만들어준 부모(클래스나 상위 객체)의 `prototype`을 가리키는 연결 고리(`__proto__`)는 반드시 존재합니다. 그 연결 고리 덕분에 우리가 클래스 없이 만든 객체도 `toString()` 같은 메서드를 사용할 수 있는 것입니다.
