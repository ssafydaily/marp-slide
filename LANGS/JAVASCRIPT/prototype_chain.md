프로토타입 체인(Prototype Chain)은 자바스크립트가 객체 지향을 구현하는 핵심 메커니즘입니다. 쉽게 말해, "나한테 없는 속성이나 메서드를 부모(상위 객체)에게 가서 빌려 쓰는 연결 고리"라고 이해하면 됩니다.

---

### 1. 프로토타입 체인의 핵심 원리

자바스크립트의 모든 객체는 내부적으로 `[[Prototype]]`이라는 숨겨진 링크를 가지고 있습니다. (코드에서는 `__proto__`로 접근 가능합니다.)

1. **탐색**: 객체의 속성에 접근할 때, 먼저 그 **객체 자신**에게 해당 속성이 있는지 확인합니다.
2. **상승**: 자신에게 없다면, `[[Prototype]]` 링크를 타고 부모 객체(프로토타입)로 올라가 확인합니다.
3. **반복**: 이를 찾을 때까지 혹은 최상위 부모인 `Object.prototype`에 도달할 때까지 반복합니다.
4. **종료**: 최상위까지 갔는데도 없다면 `undefined`를 반환합니다.

---

### 2. 알기 쉬운 예시: "가문의 가업"

상속 관계를 가업에 비유해 보겠습니다.

```javascript
// 1. 할아버지 객체 (최상위 부모)
const grandfather = {
  surname: 'Kim',
  house: 'Seoul',
  greet: function() { console.log("허허, 반갑다."); }
};

// 2. 아버지 객체 (할아버지를 프로토타입으로 지정)
const father = Object.create(grandfather);
father.job = 'Doctor'; // 아버지에게만 있는 속성

// 3. 나 객체 (아버지를 프로토타입으로 지정)
const me = Object.create(father);
me.hobby = 'Coding'; // 나에게만 있는 속성

```

이제 `me` 객체에서 여러 속성을 호출해 봅시다.

* **`me.hobby`** → "Coding" (내꺼니까 바로 출력)
* **`me.job`** → "Doctor" (나한테 없네? → **아버지**한테 가서 찾음 → 성공!)
* **`me.surname`** → "Kim" (나한테 없네? → 아버지한테도 없네? → **할아버지**한테 가서 찾음 → 성공!)
* **`me.greet()`** → "허허, 반갑다." (마찬가지로 할아버지까지 올라가서 실행)
* **`me.money`** → `undefined` (가문의 어르신 누구에게도 없음 → 실패)

---

### 3. 왜 이런 짓(?)을 하나요? (장점)

만약 프로토타입 체인이 없다면, 모든 자식 객체들은 `surname`, `house`, `greet` 같은 공통 속성을 **각자 메모리에 따로 저장**해야 합니다.

* **메모리 효율성**: 공통된 메서드나 속성을 부모(프로토타입) 한 곳에만 두면, 수만 개의 자식 객체가 생성되어도 메모리를 낭비하지 않고 공유할 수 있습니다.
* **유연성**: 부모의 속성을 수정하면 이를 상속받는 모든 자식 객체에게 즉시 반영됩니다.

---

### 4. 체인의 끝은 어디인가요?

모든 프로토타입 체인의 종점은 `Object.prototype`입니다.
우리가 평소에 객체를 만들었을 때 `toString()`, `hasOwnProperty()` 같은 메서드를 정의하지 않고도 쓸 수 있었던 이유가 바로 이 메서드들이 `Object.prototype`에 정의되어 있고, 모든 객체가 이를 **프로토타입 체인으로 연결**해서 쓰고 있기 때문입니다.

### 요약

> **내 주머니(객체)**에 없으면 **엄마 주머니(부모 프로토타입)**를 뒤지고, 거기 없으면 **할머니 주머니**를 뒤지는 것. 이 연결된 주머니들의 줄줄이 소시지가 바로 **프로토타입 체인**입니다!
