# AI Onboarding

- [예시_프로젝트] https://localhub-sc16-1.netlify.app/

#### 기획 및 설계는 ChatGpt 활용


--------------------

## 요구사항 분석 및 아이디어 도출

> 입력 프롬프트
```
너는20년차PM이야. 아래는고객사가보낸개발의뢰문서(RFP) 전문이야. 이내용을표로정리해줘.

열은[구분(필수/선택/제약), 항목, 상세내용,근거조항, 검토의견]으로 하고, 

'반드시·금지·절대＇같은강조표현이들어간조항은제약사항으로분류해줘.

---(개발의뢰문서(RFP) 원문전체붙여넣기) ---
```

--------------------

## LLM으로 선택 가능 후보 평가 하기

> 입력 프롬프트

```
우리 팀은 개발기간 3일, 개발환경은 VSCode Copilot + OpenAI API만 허용, 팀원은 신입 개발자 위주야.

아래 선택기능 후보 N개 중에서 이 조건에 맞는 것을 추천하고 이유를 설명해줘:

각 항목마다 [구현난이도 1~5, 실현가능성(난이도가 낮을수록 높음)과 임팩트(사용자 체감 효과)]를 표로 만들어줘.

--- (선택 가능한 후보 표 붙여넣기, 9 page 내용 ) ---
```

- AI가평가한 내용에 대해서 팀원과 논의후 현재 팀상황에 맞춰 점수 조정 진행

---------------------

## LLM 으로 MVP 범위 정의

> 입력 프롬프트

```
이 프로젝트의 MVP 범위를 MoSCoW 방식으로 정리해줘.

Must have는 개발 의뢰 문서(RFP)의 '필수' 항목 그대로, 

Should/Could have는 평과 결과를 고려,Won't have(이번엔 안 하는 것)도 명확히 구분해서 표로 만들어줘.

--- (개발 의뢰 문서(RFP) 원문 전체 및 평가 결과 넣기) ---
```


-----------------------

## LLM 으로 WBS 초안 작성 하기

> 프롬프트
```
위에서 확정한 MVP 범위를 바탕으로 WBS를 작성해줘. 

오늘은 YYYY-MM-DD(실제 오늘 날짜로 교체)이고 납기는 YYYY-MM-DD(실제 납기로 교체) 15시야. 

대분류는 [기획, 설계, 개발, 배포, 테스트, 문서화, 발표준비]로 나누고

각 작업마다 [작업명, 담당자(FE/BE/팀전체), 시작일, 종료일, 선행작업]을 표로 만들어줘.
```

- LLM이 작성한 내용을 팀에 맞게 재작성 및 검토

--------------------------

## 간트 차트 작성하기

- 작성한 **WBS** 를 기준으로 일정을 고려하여 업무 분담
- 일 단위, 시간 단위 자유롭게 구성

# 개발 시작

### Copilot 사용해서 작업하기

## 개발 시작 전에 강사와 미팅
- 팀이 정한 MVP / 일정이 실현 가능성이 있는지 강사님과 함께 논의 진행 
- 잘못된 방향/무리한 일정으로 세팅하고 프로젝트를 시작할 경우,프로젝트를 완성하기 어려울 수 있음

---------------------------

## 개발 시작 전 주의 사항

- 팀 프로젝트는 AI를 활용할 때, **Ask** 모드만 사용하는 것으로 제한
  - Ask 모드: AI가 사용자의 질문에 응답만 해주는 모드
- 처음 다루는 기술로 프로젝트를 진행할 때는 차근차근 안내해달고 요청하고, 천천히 따라하기

<hr>

**[참고] Ask 모드 사용 이유**
> - Agent 모드는 AI 1 강의에서 진행 예정
> - 문답을 통해 코드의 작성 흐름 파악, 모르는 내용을 학습 목록으로 가져가기
> - 토큰은 반별로 공유되기 때문에 서로 배려하면서 사용
>   - Agent 모드는 AI가 다단계 작업을 자동을 수행하기 때문에 과하게 토큰을 사용하는 경우가 있음
