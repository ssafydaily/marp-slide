# 기초 통계량

- 데이터를 표현하는 변수들은 수많은 다른 값들을 가진다.
- 어떻게 시작할까?

- 데이터 분석의 첫 단계는 데이터의 크기, 분포, 중심, 퍼짐을 파악하기
  - 모든 값들을 확인하는 것은 불가능

### **=> 대표값(위치)과 변동성(퍼짐)을 요약하는 통계량 활용**

---

# 기초 통계량 : 위치

## 각 변수를 대표하는 값
- 위치를 추정하는 다양한 값
  - 평균(mean): 전체적인 중심을 나타내지만 이상치에 민간
  - 절사평균(trimmed mean): 상하위 극단 값 일부를 제거. 이상치 영향을 줄이는 데 유용
  - 가중평균(weighted mean): 값의 중요도(가중치)를 반영. 값의 영향력이 동일하지 않을 때
  - 중간값(median): 정렬된 데이터의 중앙 위치. 왜곡된 분포에서 안정적으로 중심을 나타냄
  - 가중 중간 값(weighted median): 값의 가중치를 고려한 중앙 위치. 표본 비중이 다른 조사에서 활용
  - 백분위수(percentile): 데이터를 100등분 했을 때 특정 위치 값. 분포의 형태와 위치를 파악하는데 사용

---

# 기초 통계량 : 변이

- 데이터가 얼마나 퍼져 있는지를 확인
- 분산(variance): 평균에서 얼마나 퍼져 있는지를 나타내는 지표
- 표준편차(standard variance): 데이터의 편차(각 값에서 평균을 뺀 값)의 기준이 되는 값
- 사분할 범위(IQR): 25번째와 75번째 백분위수의 차이를 보는 것

---

# 기초 통계량의 활용
- 이상치 파악
  - 이상치는 평균/분산을 크게 왜곡
  - 예) 평균 몸무게가 60kg 인데, 120kg이 포함되어 있는 경우
- 비교와 예측 가능
  - 두 개 이상의 그룹을 비교할 때 **어떤 차이** 가 있는지 시각적으로 확인
  - 예) 남학생과 여학생의 키 분포 비교
- 적절한 분석 방법 선택
  - 데이터의 분포 형태에 따라 **평균, 중앙값 등을 선택** 하여 분석 가능
  - 대칭형 분포라면 평균 사용, 왜곡된 분포면 중앙값이 더 적절
  - 예) 연봉/집값처럼 오른쪽 꼬리가 긴 분포는 중앙값 사용

# 조건부 확률 및 베이즈 정리
- 전통적인 통계가 `불변의 확률`을 계산한다면, 베이즈 정리는 **`업데이트되는 확률`** 을 다룹니다.

### 현실에서 베이즈 정리가 필요한 이유

① 역방향의 추론이 가능하게 함 (결과에서 원인으로)
  - 현실에서는 '원인'을 알 때 '결과'가 나올 확률은 알기 쉽지만, 반대로 '결과'를 보고 '원인'을 맞혀야 하는 경우가 훨씬 많습니다.

② 불완전한 정보의 보완 (사전 지식의 활용)
  - 데이터가 적은 상황에서도 우리가 이미 알고 있는 지식(사전 확률)을 계산에 포함시킬 수 있음
  - 새로운 데이터가 한 건씩 들어올 때마다 확률을 실시간으로 업데이트하여 점점 정답에 가까워질 수 있음

--- 

- 알기 쉬운 것: 감기에 걸렸을 때(원인) 기침을 할(결과) 확률
  - 알고 싶은 것 (베이즈 정리): 기침을 하는데(결과) 이게 감기일(원인) 확률

- 추천 시스템 => 행동을 관찰
  - A: 사용자가 어떤 영화를 클릭했다
  - B: 사용작 실제로 좋아하는 장르는?
    - P(행동(A) | 취향(B)) 은 모델링이 쉽다
    => 행동 로그 기반으로 원하는 건 P(취향(B)| 행동(A))

--- 
- 직접 구하기 어려운 P(A|B) 를, 구하기 쉬운 P(B|A), P(A), P(B)로 구한다.
- 조건부 확률의 방향을 뒤집어 주는 역할
- 현실에서 측정 가능한 확률: A -> B, 알고싶은 확률: B -> A

$$
  P(A|B) = \frac{P(A \cap B)}{P(B)}
  
$$

$$
  P(A|B) = \frac{P(B|A)P(A)}{P(B)}

$$

# 나이브 베이즈 분류

## 스팸 메일 판단

- 스팸 메일과 정상 메일에서 특정 단어가 등장한 비율을 관찰

$$
P(스팸|단어) = \frac{P(단어|스팸)P(스팸)}{P(단어)}
$$


---

```python
def calculate_spam_probability(word_in_spam, word_in_ham, total_spam, total_ham):
    """
    word_in_spam: 스팸 메일 중 해당 단어가 포함된 메일 수
    word_in_ham: 정상 메일 중 해당 단어가 포함된 메일 수
    total_spam: 전체 스팸 메일 수
    total_ham: 전체 정상 메일 수
    """
    
    # 1. 사전 확률 (Prior Probabilities)
    p_spam = total_spam / (total_spam + total_ham)
    p_ham = total_ham / (total_spam + total_ham)
    
    # 2. 우도 (Likelihood) - 해당 단어가 각 메일군에서 나타날 확률
    # (단어가 한 번도 안 나왔을 경우 분자가 0이 되는 것을 방지하기 위해 0.01을 더해주는 '라플라스 스무딩'을 적용할 수 있으나, 여기선 기초 계산만 수행)
    p_word_given_spam = word_in_spam / total_spam
    p_word_given_ham = word_in_ham / total_ham
    
    # 3. 전체 단어 등장 확률 P(Word)
    # P(Word) = P(Word|Spam)P(Spam) + P(Word|Ham)P(Ham)
    p_word = (p_word_given_spam * p_spam) + (p_word_given_ham * p_ham)
    
    if p_word == 0:
        return 0
    
    # 4. 베이즈 정리 적용: P(Spam|Word)
    p_spam_given_word = (p_word_given_spam * p_spam) / p_word
    
    return p_spam_given_word

# --- 가상의 데이터 설정 ---
# 전체 메일 100통 중 스팸 40통, 정상 60통
# '광고'라는 단어가 스팸 메일 32통에 들어있고, 정상 메일 6통에 들어있을 때
target_word = "광고"
prob = calculate_spam_probability(word_in_spam=32, word_in_ham=6, total_spam=40, total_ham=60)

print(f"'{target_word}' 단어가 포함된 메일이 스팸일 확률: {prob:.2%}")
```
