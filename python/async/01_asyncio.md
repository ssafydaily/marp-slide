# 비동기 I/O 살펴보기

- 프로그래밍과 시스템 디자인에서 동시성(Concurrency)과 병렬성(Parallelism)은 자주 혼용되지만, 핵심적인 차이가 있다. 

## 1. 동시성 vs 병렬성: 핵심 비교

### 동시성 (Concurrency)

* **개념:** 여러 작업을 **동시에 다루는 것**입니다. (Dealing with many things at once)
* **특징:** 실제로 같은 시간에 실행되는 것이 아니라, CPU가 여러 작업을 아주 빠르게 번갈아 가며 처리(Context Switching)하여 **동시에 실행되는 것처럼 보이게** 만듭니다.
* **목적:** 유휴 시간(Idle time)을 최소화하여 논리적인 효율성을 높이는 것입니다. 싱글 코어에서도 가능합니다.

---------

### 병렬성 (Parallelism)

* **개념:** 여러 작업을 **실제로 동시에 실행하는 것**입니다. (Doing many things at once)
* **특징:** 물리적으로 여러 개의 코어(Multi-core)가 각자 다른 작업을 같은 시점에 수행합니다.
* **목적:** 물리적인 계산 속도를 높여 작업 완료 시간을 단축하는 것입니다.

---------

| 구분 | 동시성 (Concurrency) | 병렬성 (Parallelism) |
| --- | --- | --- |
| **핵심 키워드** | 구조, 논리적 개념, 시분할 | 실행, 물리적 개념, 멀티코어 |
| **CPU 코어** | 싱글 코어에서도 가능 | 반드시 멀티 코어 필요 |
| **관점** | 작업을 어떻게 "관리"하는가? | 작업을 어떻게 "수행"하는가? |

---------

## 2. 비동기 I/O (Asynchronous I/O)와의 연관성

- 비동기 I/O는 **동시성**을 구현하는 아주 효율적인 전략 중 하나

### 왜 비동기 I/O가 필요한가?

- 일반적인 **동기(Blocking) I/O** 환경에서는 파일을 읽거나 네트워크 요청을 보낼 때, 응답이 올 때까지 CPU는 아무것도 하지 않고 기다린다. 

#### ==> 이는 엄청난 자원 낭비.

---------

### 비동기 I/O의 작동 방식

1. **요청 후 즉시 복귀:** 작업을 던져놓고 결과가 나올 때까지 기다리지 않고 다음 코드를 실행합니다.
2. **이벤트 기반:** 작업이 완료되면 운영체제가 알림을 주거나, 콜백(Callback) 함수를 호출합니다.
3. **효율성:** CPU는 데이터가 도착하기를 기다리는 대신, 그 시간에 다른 작업을 처리할 수 있습니다.

-----------

### 동시성/병렬성과의 관계

* **비동기 I/O와 동시성:** 비동기 I/O는 싱글 스레드 환경(예: Node.js)에서도 수만 개의 연결을 처리할 수 있다. 
  * 이는 물리적으로 병렬 실행되는 것은 아니지만, 대기 시간을 없앰으로써 **높은 동시성**을 확보한다.
* **비동기 I/O와 병렬성:** 비동기로 요청한 작업들이 커널 수준이나 별도의 워커 스레드에서 실제로 동시에 처리된다면, 이는 **병렬성**의 도움을 받아 전체 성능을 가속화하는 구조가 된다.

---

## 3. 요약: 비유로 이해하기

* **동기(Blocking):** 점원이 주문을 받고 음식이 나올 때까지 손님 앞에 서서 기다리는 상황 (다른 주문 못 받음).
* **병렬성:** 점원이 4명이라서 4명의 손님 주문을 동시에 요리하는 상황.
* **비동기 + 동시성:** 점원은 1명이지만, 주문만 받고 진동벨을 준 뒤 바로 다음 손님 주문을 받는 상황. 요리가 나오는 동안 점원은 쉬지 않고 계속 주문을 처리.

> 결국 **비동기 I/O**는 적은 자원(스레드)으로도 많은 작업을 동시에 처리하는 것처럼 보이게 만드는 **동시성** 설계의 핵심 기술.

---------

## 파이썬에서 동시성과 병렬성

- 비동기 루틴은 결과를 기다리는 동안 중지 되고, 다른 루틴이 실행될 수 있도록 한다.
- 비동기 코드는 비동기 루틴들을 조율하여 작업들의 동시 실행을 가능하게 한다.
- 흰색 항은 개념을, 녹색 항은 개념의 구현 방식을 나타낸다.

# 비동기 I/O
<br>
- 체스 마스터 주디트 폴가르는 여러 아마추어 선수들과 대결하는 체스 시범 경기를 개최
- 그녀는 동기식 과 비동기식 , 두 가지 방식으로 시범 경기를 진행

> 가정:
> - 24명의 상대
> - 주디트는 체스에서 한 수를 두는 데 5초, 상대방은 각자 55초 안에 움직여야 한다.
> - 게임당 평균 30쌍의 움직임(총 60수)이 있다.
> - 동시 진행 방식 : 주디트는 게임이 끝날 때까지 한 번에 한 게임씩만 플레이(두 게임을 동시에 플레이 않함)
> - 각 게임은 (55 + 5) * 30 == 1800 초, 즉 30분 소요.
> - 전체는 24 * 30 == 720 분, 즉 12시간이 소요.

-----------------
## 비동기 방식
> - 주디트는 테이블을 옮겨 다니며 각 테이블에서 한 수씩 둡니다. 
> - 테이블을 떠나면 상대방이 기다리는 동안 다음 수를 둡니다. 
> - 24개의 게임 모두에서 한 수씩 두는 데 주디트는 24 * 5 == 120 초, 즉 2분이 소요. 
> - 이제 전체 전시 경기는 120 * 30 == 3600 초, 즉 단 1시간 으로 단축됩니다.

# Python에서 비동기 I/O

## 코루틴과 코루틴 함수
<br>

- 비동기 I/O의 핵심은 **코루틴(coroutine)** 이라는 개념.
- 코루틴은 실행을 일시 중단했다가 나중에 다시 시작할 수 있는 객체. 
- 코루틴은 제어권을 이벤트 루프로 넘겨 다른 코루틴이 실행될 수 있다. 
- 코루틴 객체는 코루틴 함수(비동기 함수)를 호출하여 생성된다. 
- 코루틴은 ` async def` 구문을 사용하여 정의합니다.

-------------------

## 동기 코드 예시

```python
import time 

def count():
    print('One')
    time.sleep(1)
    print('Two')
    time.sleep(1)

def main():
    for _ in range(3):
        count()

start = time.perf_counter()
main()
elapsed = time.perf_counter() - start
print(f'{__file__} 실행 시간 {elapsed:0.2f} 초.')
```

-------------------

## 비동기 예시

```python
import time 
import asyncio

async def count():
    print('One')
    await asyncio.sleep(1) # <-- asyncio.sleep()
    print('Two')
    await asyncio.sleep(1) # <--

async def main():
    # ---- asyncio.gather()
    await asyncio.gather(count(), count(), count())


start = time.perf_counter()
# ---- asyncio.run()
asyncio.run(main())
elapsed = time.perf_counter() - start
print(f'{__file__} 실행 시간 {elapsed:0.2f} 초.')
```
-------------

## 비동기 예시

- `async/await` 키워드를 사용하여 `print`를 출력하고 1초간 기다린 다음 `print`를 출력 하고 다시 1초간 기다리는 코루틴 함수 `count()`
- `await` 키워드는 `asyncio.sleep()` 함수의 실행을 기다리게 한다. 
- 이렇게 하면 프로그램의 이벤트 루프에 제어권을 되돌려주면서 "나는 1초 동안 대기할 테니, 그동안 다른 작업을 실행해도 된다"라고 말하는 것과 같다.

- `main()` 함수는 `asyncio.gather()`로 세 개의 `count()` 인스턴스를 동시에 실행하고 결과를 기다린다.  
- `asyncio.run()`으로 이벤트 루프와 `main()` 함수를 실행한다.
